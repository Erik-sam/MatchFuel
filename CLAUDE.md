# MatchFuel — Football Performance Optimizer
*(работно заглавие, може да се смени)*

## Какво е проектът
Уеб приложение за футболисти, което генерира персонализиран
хранителен и загрявъчен план преди мач, на база лични данни,
тренировъчен режим и история на представянето.

## Технически детайли
- Чисти статични файлове — без Node.js, без build стъпка
- Всички JS файлове са ES modules (type="module")
- Chart.js се зарежда от CDN в index.html
- Данните се пазят в localStorage
- Няма backend — всичко е client-side

## Файлова структура
Целият frontend код живее в подпапката `MatchFuel/`
(преди се казваше `backend/` — преименувана 2026-05-07).
Отвори `MatchFuel/index.html` директно в браузъра — няма build стъпка.

```
MatchFuel/
├── index.html        → главна страница (зарежда само ui.js)
├── style.css         → стилове (вкл. PRO modal, feedback, recovery)
├── ui.js             → входна точка, свързва HTML с логиката
├── mealPlanner.js    → генерира хранителния план и timeline
├── nutrition.js      → foodDatabase + calculateMacros
├── foodMatcher.js    → fuzzy matching, BG/EN aliases, category fallback (feature 1)
├── energy.js         → calculateEnergyNeeds + recency-decay weekly load
├── energyChart.js    → Chart.js графа на енергийното ниво
├── warmup.js         → generateWarmup + exerciseLibrary + injury filter (feature 7)
├── feedback.js       → adaptive learning, корекционни множители
├── bodyFat.js        → body fat % калкулатор + скала UI (feature 2)
├── hydration.js      → daily water + timeline reminders (feature 4)
├── recovery.js       → post-match refuel meal + light exercises (feature 8)
└── profiles.js       → localStorage profiles + history (feature 10)
```

## Текущо реализирани функции
- Въвеждане на тегло, сън, час на мач
- Седмични тренировки (тип + минути)
- Вечерно хранене (вход на храни с грамове)
- Генериране на timeline: вечеря → закуска → предматчево → загрявка → мач
- Енергийна графа (Chart.js) с маркери за събитията
- Загрявка с упражнения по категории (mobility, activation, cardio, football)
- GIF полета за упражненията (gifUrl в warmup.js)
- Фийдбак форма след мач (sleek тъмен дизайн)
- Adaptive learning — корекции в следващия план на база фийдбак
- Прогрес графа на представянето

## Planned features — приоритетен ред

### 1. Разширена база данни с храни + fuzzy matching
- Разшири foodDatabase от ~10 до ~60+ храни
- Добави fuzzy matching: "chiken" → "chicken", "пилешко" → "chicken"
- Добави категорийно fallback: ако храната е непозната →
  класифицирай по категория (месо/зеленчук/плод/зърно) и
  използвай средните макроси за категорията
- Autocomplete menu при въвеждане на храна в input полетата
- Проверка дали въведената храна е реална (валидация)

### 2. Разширен потребителски профил
- Добави полета: височина (cm), възраст, пол (мъж/жена)
- Body fat calculator на база тегло + височина + пол + възраст
- Визуализация на body fat индекса (скала + маркер "ти си тук")
- Тегло в ден на мача: преди закуска и след мача
- Час на ставане — закуската в timeline се изчислява от него
  (до 1 час след ставане), с опция "пропусни закуска"
- Позиция на играча (вратар, защитник, полузащитник, нападател, съдия)
- За колко минутен мач се подготвяш (1 до 130)

### 3. Диетични предпочитания
- Добави опция: обикновен / вегетарианец / веган / без глутен
- Това променя proteinSources и carbSources в mealPlanner.js
- Примери: веган → tofu, legumes, tempeh вместо chicken/fish/egg

### 4. Хидратация
- Добави секция за течности в timeline
- Изчислявай препоръчително количество вода за деня
  на база тегло, температура (сезон) и интензивност
- Напомняния на всеки 1-2 часа в timeline-а

### 5. Гъвкавост на храненето спрямо часа на мача
- Ако мачът е след 18:00 → добави обяд в timeline
- Зависимост: последното хранене преди мача да е
  не по-малко от 2.5h и не повече от 4h преди
- Адаптирай порциите спрямо времето между храненията

### 6. Последните 3 тренировки (вместо седмични)
- Замени "тренировки тази седмица" с "последни 3 тренировки"
- Всяка тренировка: дата, тип, минути
- Изчислявай умората на база давността (вчера = повече умора
  от преди 5 дни)

### 7. Информация за контузии
- Input поле за активни контузии (свободен текст или списък)
- Ако има контузия → питай: "По-лека загрявка или пълна?"
- Адаптирай exerciseLibrary — пропускай упражнения за
  засегнатата зона

### 8. Възстановяване след мач
- След фийдбака добави секция "Post-match recovery"
- Препоръчи: хранене за възстановяване (протеин + въгл.
  в рамките на 30 мин след мача)
- Препоръчи: упражнения за възстановяване (stretching,
  foam rolling, light walk)

### 9. Pro версия (UI само — без реален backend)
- Добави "Upgrade to Pro" бутон с диамантче
- Pro model: показва кои устройства са налични
  (Google Fit, Apple Health, Garmin, Polar)
- Бутон за качване на снимка на сърдечен ритъм
  (UI само, с надпис "Available in Pro")
- Real-time сърдечен ритъм мониторинг (заглушен за free)

### 10. Многостраничност / профили
- Раздели на: Профил / План / История
- localStorage профили — всеки потребител си има акаунт
  (без backend — просто localStorage с имена)
